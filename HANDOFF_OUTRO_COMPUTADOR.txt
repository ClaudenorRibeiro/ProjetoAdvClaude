SEÇÃO 2 — VISÃO GERAL DO PROJETO
================================================================

Sistema de gestão para escritório de advocacia.
Substitui planilhas Excel. Áreas: Trabalhista, Previdenciária
e Família (SEM Criminal e Tributário).

Escritório atual: ~12 advogados
Capacidade: até 500 usuários por instância

STACK TÉCNICO:
  Backend:  Node.js + Express (porta 3001)
  Frontend: React + Vite (porta 3000)
  Banco:    MySQL 8.0.46
  Auth:     JWT (localStorage)
  HTTP:     Axios com interceptors

ESTRUTURA DE PASTAS:

================================================================

Banco: sistema_advocacia (MySQL)
Conexão: pool com mysql2/promise, pool.execute() para prepared statements

PADRÃO DE TRANSAÇÃO:
  const conn = await pool.getConnection();
  await conn.beginTransaction();
  try { ... await conn.commit(); }
  catch { await conn.rollback(); throw err; }
  finally { conn.release(); }

SOFT DELETE: campo ativo = 0 (nunca deleta fisicamente)

ESTRUTURA DO BANCO:

SEÇÃO 5 — AUTENTICAÇÃO E PERMISSÕES
================================================================

HIERARQUIA:
  Nível 0 → Superusuário (superadmin) — invisível, dono do sistema
  Nível 1 → Administrador             — acesso total ao escritório
  Nível 2 → Usuários normais          — permissões configuradas

BACKEND:
  Middleware autenticar: verifica JWT
  Middleware apenasAdmin: nivel <= 1
  verificarPermissao(modulo, acao): checa tabela permissoes
  Admins (nivel <= 1) têm acesso irrestrito — sem checar permissoes

FRONTEND (AuthContext):
  const { usuario, ehAdmin, temPermissao } = useAuth();
  ehAdmin → nivel <= 1
  temPermissao('modulo', 'acao') → verifica permissão específica
  Admins sempre retornam true em temPermissao

MÓDULOS DE PERMISSÃO (array em Configuracoes.js → MODULOS_PERM):
  'pessoas', 'processos', 'pastas', 'prazos', 'tarefas',
  'audiencias', 'pericias', 'financeiro', 'documentos',
  'publicacoes', 'relatorios'

AÇÕES: 'visualizar', 'cadastrar', 'alterar', 'excluir'

PERMISSÃO ESPECIAL: 'pastas.alterar' → renumerar pasta

SEÇÃO 9 — CAMPOS DO BANCO — tblForum e tblVara
================================================================

tblForum:
  id, abrev_nome, nome*, cidade, uf, cep, logradouro,
  num_end, compl_end, bairro, ativo, criado_por, criado_em,
  alterado_por, alterado_em

tblVara:
  id, abrev_nome, forum_id*, nome*, codVaraNoProc (VARCHAR 15),
  compl_end, tel, email, ativo, criado_por, criado_em,
  alterado_por, alterado_em

codVaraNoProc: segmento J+TT+OOOO do número CNJ — 7 dígitos.
  Ex: vara 1 do TRT-2 SP = "5020001"
  Padrão Fórum Ruy Barbosa: "50200NN" (NN = número da vara, 2 dígitos)

tblTipoProc:
  id, codTipoProc (VARCHAR 1), nome*, ativo, criado_por...
  codTipoProc = dígito J do CNJ (ex: '5' = Trabalhista)
  PENDENTE: popular os valores de codTipoProc no banco.
