3. CONFIGURAR A CHAVE .PEM (autenticação sem senha)
--------------------------------------------------------------------------------

O servidor AWS usa autenticação por chave privada (.pem), não por senha.

3.1. Na tela de Login do WinSCP, clique em "Avançado..." (ou "Advanced...")

3.2. No menu lateral esquerdo, vá em:
     SSH → Autenticação  (ou  SSH → Authentication)

3.3. No campo "Arquivo de chave privada" clique no botão de pasta (...)

3.4. Localize o arquivo .pem no seu computador
     (Ex: C:\Users\SeuUsuario\Downloads\AntonioAdv.pem)

3.5. O WinSCP perguntará se deseja converter o arquivo para o formato .ppk
     Clique em "OK" / "Converter" — isso é necessário para o WinSCP funcionar
     Salve o arquivo .ppk no mesmo local do .pem

3.6. O campo ficará preenchido com o caminho do arquivo .ppk
     Clique em OK para fechar a janela Avançado

3.7. De volta à tela de Login, clique em "Salvar" para guardar esta configuração
     Dê um nome como "AntonioAdv" e clique em OK

--------------------------------------------------------------------------------
5. LOCALIZAR A PASTA DO SISTEMA NO SERVIDOR
--------------------------------------------------------------------------------

O sistema NÃO fica em /home/ubuntu/ — fica em /var/www/advocacia/

Para navegar até lá:

OPÇÃO A — Pelo painel direito do WinSCP:
  1. Clique no campo de endereço (barra superior do painel direito)
  2. Digite: /var/www/advocacia
  3. Pressione Enter

OPÇÃO B — Se não achar pelo WinSCP, use o terminal do Lightsail:
  1. Acesse o terminal no painel da AWS Lightsail
  2. Digite o comando abaixo para localizar o arquivo .env:
     find / -name ".env" -path "*/backend/*" 2>/dev/null
  3. O resultado será: /var/www/advocacia/backend/.env
  4. Isso confirma que a pasta do sistema é: /var/www/advocacia/

ESTRUTURA DE PASTAS DO SISTEMA:
  /var/www/advocacia/
  ├── backend/          ← API Node.js
  │   ├── .env          ← Configurações sensíveis (SMTP, banco, JWT)
  │   ├── src/
  │   └── ...
  └── frontend/         ← Interface React (já compilada)
      └── dist/


--------------------------------------------------------------------------------
6. EDITAR O ARQUIVO .ENV (exemplo: trocar a senha do e-mail SMTP)
--------------------------------------------------------------------------------

1. No painel direito do WinSCP, navegue até: /var/www/advocacia/backend/
2. Localize o arquivo chamado ".env"
   (se não aparecer, vá em: Opções → Preferências → Painéis → marcar "Mostrar arquivos ocultos")
3. Clique duas vezes no arquivo .env para abrir no editor interno do WinSCP
4. Faça a alteração desejada. Exemplo — trocar a senha SMTP:
   Linha original:  SMTP_PASS=senhaantiga
   Linha nova:      SMTP_PASS=novasenha
5. Salve o arquivo: Ctrl+S ou Arquivo → Salvar
6. O WinSCP perguntará se deseja enviar o arquivo de volta ao servidor — clique em "Sim"


--------------------------------------------------------------------------------
7. REINICIAR O BACKEND APÓS ALTERAR O .ENV
--------------------------------------------------------------------------------

Depois de salvar o .env, o backend precisa ser reiniciado para carregar as novas
configurações. Faça isso pelo terminal do Lightsail (ou qualquer terminal SSH):

  pm2 restart advocacia-backend --update-env

O flag --update-env é OBRIGATÓRIO — sem ele, o PM2 não recarrega as variáveis
do arquivo .env, e a alteração não surte efeito.

Para confirmar que reiniciou com sucesso:
  pm2 list

O processo "advocacia-backend" deve aparecer com status "online" e uptime zerado
(ex: "0s"), indicando que acabou de reiniciar.


--------------------------------------------------------------------------------
8. DICAS IMPORTANTES
--------------------------------------------------------------------------------

- O arquivo .pem é a "chave" do servidor. Guarde-o em lugar seguro.
  Sem ele, não é possível acessar o servidor.

- Nunca compartilhe o arquivo .pem ou o conteúdo do .env com ninguém.

- O WinSCP salva a configuração da conexão — na próxima vez, basta abrir
  o WinSCP e clicar em "Login" na conexão salva "AntonioAdv".

- Se o arquivo .env não aparecer no WinSCP:
  Menu: Opções → Preferências → Painéis → "Mostrar arquivos ocultos" ✓

- Para ver os logs do sistema em tempo real (útil para diagnóstico):
  pm2 logs advocacia-backend

- Para ver apenas os últimos erros:
  pm2 logs advocacia-backend --err --lines 50
